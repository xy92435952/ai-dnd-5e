DM Agent 四层架构

当前 DM 流程按 input_layer、rules_layer、 narrative、memory_layer 四个维护边界组织。 核心目标是让 AI 负责故事表达和意图理解，让后端规则层负责可验证的 5e 裁定、输入安全和状态写入。

核心原则

这四条是后续改 prompt、规则、战斗接口和记忆策略时的设计底线。
AI 负责表达 Narrative

DM LLM 可以解释意图、塑造氛围、给出选择，但不能直接绕过后端规则数学来改 HP、金币、资源和战斗结果。

主流程

从玩家行动到前端响应的完整链路，叙事分支之前会先通过输入、安全、骰池和规则上下文。
/game/action
接收玩家行动
API 层组装 session、game_state、module_context 和 action_source。
input_layer
输入归一化
识别行动来源，并用 input_guard 拦截离题、作弊和 prompt 注入。
pre_roll_dice
预生成骰池
提供 d20、adv、dis、d4 等骰池，避免叙事层临场自由编骰。
rules_layer
规则上下文
说明合法规则术语、越权边界、战斗触发和检定裁定提示。
memory_layer
记忆上下文
合并战役状态、近期历史、RAG 片段和关键事件，供 DM 使用。
explore_dm / combat_dm
叙事裁定
按 combat_active 分流，探索推进故事，战斗处理回合与 AI 单位。
parse_validate
JSON 校验
修复和补齐 LLM 输出字段，保证下游响应结构稳定。
StateApplicator
状态写库
应用 state_delta、combat_update、日志和会话历史，返回 PlayerActionResponse。
blocked route
拒绝分支
当 verdict 为 off_topic、rule_violation 或 injection 时，直接返回兼容结构的拒绝叙事，不进入规则和记忆，不污染 checkpoint。
combat route
战斗分支
combat_active 为 true 时进入 combat_dm；自然语言战斗会先经 action_parser 转成 move、attack、spell、creative 等结构化动作。

规则层

规则层的重点是“允许合理玩法，拦截越界宣告”。这次修复后的关键点是优势骰和激励骰不再被误伤。
应放行
请求进行技能检定，例如调查、潜行、说服、察觉、洞察。
声明使用优势骰、劣势骰、帮助动作、祝福 d4、吟游激励骰。
询问自身 HP、资源、法术位、装备或当前规则状态。
由 DM 自己生成并被玩家点击的合理选项。
应拦截
直接宣告自动命中、自动暴击、跳过豁免或无需检定。
要求给角色加满 HP、凭空获得大量金币/经验或神器。
修改敌人 AC/HP/属性，或瞬间跳到最终战。
与游戏无关的现实问题、代码请求、新闻天气等内容。

四类 verdict 决定是否继续进入 DM Agent 主链路。
{
  "verdict": "in_game",
  "reason": "合法规则术语",
  "refusal": ""
}

{
  "verdict": "off_topic",
  "reason": "明显与跑团无关",
  "refusal": "请用游戏内行动继续冒险"
}

自然语言战斗

战斗体验修复的核心是让“玩家意图”先变成结构化动作，再由真实执行结果决定叙事。
01
本地常见意图解析 action_parser 优先识别移动、近战、远程、最近敌人、身旁敌人等高频表达，稳定且低延迟。
02
LLM parser 补足复杂表达 当本地规则无法确定时，LLM parser 尝试把自然语言整理成 actions 数组，但仍必须落入后端支持的动作类型。
03
fallback 保证不中断 LLM 失败或超时时使用保守 fallback。若近战目标不可达，本轮只移动靠近，不伪造一次范围外攻击。
04
真实执行结果驱动叙事 API 根据实际 executed_action_types 选择 move、attack、creative、out_of_range 等叙事类型，避免“只是移动却讲成攻击失败”。

记忆层

记忆层只整理上下文，不做规则覆盖。它让故事连续，但不替代战斗和状态结算。

维护热点

后续继续拆分时，优先沿着这些边界移动代码。
backend/services/graphs/dm_agent.py：LangGraph 节点、prompt、规则和记忆上下文。
backend/services/input_guard.py：离题、作弊、注入、合法规则术语分类。
backend/services/action_parser.py：自然语言战斗动作解析。
backend/services/state_applicator.py：DM 输出落库和会话状态更新。
backend/api/combat/*、combat_service.py、dnd_rules.py：确定性战斗规则。
frontend/src/pages/Adventure.jsx 与 frontend/src/components/combat/：前端展示和交互入口。
